HTML, CSS, JavaScript
VERSIONES

# Introducción

¿Te has preguntado alguna vez cómo algunas aplicaciones y sitios web te permiten consultar el tiempo?

En este tutorial, vamos a crear una aplicación web que muestre el clima actual, utilizando HTML, CSS, JavaScript y un ingrediente especial... ¡APIs!

Gracias a la potencia de Open Weather y a un poco de magia de programación, podemos consultar la previsión meteorológica local.

Al finalizar este tutorial, habrás aprendido a:

  • Obtener datos asíncronos con una clave API.
  • Organizar el contenido con el modelo de diseño CSS (Flexbox).
  • Cambiar el contenido de la página renderizada dinámicamente.

# Paso 1: Configuración del proyecto

Primero, necesitaremos crear las carpetas y archivos que contendrán el código de nuestro proyecto. Comencemos abriendo una terminal y creando una nueva carpeta llamada weather-app usando el comando mkdir. Luego, accede a esta carpeta usando el comando cd:

mkdir weather-app
cd weather-app
touch index.html styles.css script.js

Esto creará los siguientes archivos en la carpeta weather-app:

  • Un archivo index.html para renderizar la aplicación.
  • Un archivo styles.css para personalizar la apariencia de nuestra aplicación.
  • Un archivo script.js donde obtendremos nuestros datos meteorológicos.

Volveremos a estos archivos más adelante. Pero por ahora, ¡vayamos a OpenWeather!

# Paso 3: Escribir HTML

¡Es hora de empezar a escribir código!

Volvamos a nuestro editor de código y abramos el archivo index.html.

Para empezar, añade el siguiente código HTML:

<!DOCTYPE html>
<html lang="en">
<head>
  <link href="styles.css" rel="stylesheet" />
  <title>Weather App</title>
</head>
<body>

</body>
</html>

Además de la declaración <!DOCTYPE html> necesaria, hemos incluido un elemento <link> en el <head> para conectar nuestro HTML con nuestro archivo styles.css (en el que trabajaremos en el siguiente paso).

A continuación, agreguemos el siguiente código HTML dentro del elemento <body>:

<body>
  <main>
    <section id="weather-wrapper">
      <h1>Weather App</h1>
      <div id="weather-search">
        <input id="search" type="text" placeholder="Search by city" />
        <input id="submit" type="submit" onclick="fetchWeather()" value="Search" />
      </div>
      <div id="weather-data" style="display: none;"></div>
    </section>
  </main>
  <script src="script.js"></script>
</body>

El elemento <main> contiene toda la estructura de nuestra aplicación e incluye un elemento <section> con lo siguiente:

  • Un elemento de encabezado <h1>.
  • Un elemento <div> con dos componentes <input> para buscar el clima por ciudad.
  • El otro campo <div> está vacío por ahora, pero se rellenará con los datos que devuelva la función fetchWeather() que escribiremos más adelante.

También añadimos un elemento <script> para conectar nuestro HTML con el archivo script.js.

Nota: El elemento <script> se escribe antes de la etiqueta de cierre </body> para garantizar que todo el HTML se cargue y sea accesible para JavaScript.

# Paso 4: Escribir CSS

¡Vamos a darle un toque especial a nuestra aplicación del tiempo dándole estilo con CSS!

En la mayor parte de este paso, utilizaremos CSS estándar. Sin embargo, también usaremos un modelo de diseño interesante para CSS llamado Flexbox para organizar el contenido de nuestra aplicación.

Dentro de nuestro archivo styles.css, comencemos agregando lo siguiente:

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  background-image: url(https://bit.ly/4bcK1Hc);
  background-repeat: no-repeat;
  background-size: cover;
  font-family: 'Gill Sans', 'Gill Sans MT', Calibri, 'Trebuchet MS', sans-serif;
  height: 100vh;
}

Esto es lo que hicimos en el fragmento anterior:

  • Primero, seleccionamos todo (es decir, *), estableciendo por defecto margin y padding a cero, y factorizando el padding y border en tamaños de elementos con box-sizing: border-box.
  • Luego, hacemos que el body tenga la misma altura que la pantalla en la que se renderiza con 100vh. También configuramos su información font-family, con una fuente sans-serif predeterminada.
  • Por último, configuramos background-image pasando un enlace de imagen de Brooklyn (donde tiene su sede Codedex) a la función url(). ¡Siéntete libre de usar tu propia imagen!

A continuación, vamos a seleccionar el elemento <main> y añadirle los siguientes estilos:

main {
  background-color: rgba(227, 193, 173, 0.85);
  position: relative;
  top: 30%;
  border: 1px solid;
  border-radius: 5px;
  min-width: 540px;
  max-width: 50%;
  margin: auto;
  padding: 2em;
}

h1 {
  margin-bottom: 20px;
}

Aplicamos el siguiente estilo al elemento <main>:

  • Añadimos un valor RGB para background-color, así como un cuarto valor (alfa) que hace que el elemento sea un 85% "opaco" (o 15% transparente).
  • Se le asignó una posición relative y se colocó al 30% del borde top de la pantalla.
  • Se agregaron algunas configuraciones de modelo de caja, incluyendo border, border-radius y padding.
  • Le dimos un tamaño mínimo min-width de 540 píxeles y un ancho máximo max-width del 50%.
  • Dado que el elemento <main> es un elemento de bloque, lo centramos con margin: auto.

Por último, añadimos algo de espacio debajo del elemento h1 con margin-bottom.

Guardemos nuestro archivo styles.css. En este punto, nuestra aplicación del tiempo renderizada debería verse así:

[Imagen: Aplicación centrada con fondo transparente sobre la imagen de Brooklyn]

Ahora, hagamos que los elementos <input> parezcan más grandes para que coincidan mejor con el encabezado <h1> que está encima de ellos. Selecciona estos elementos con #search y #submit, y agrega lo siguiente:

#search {
  border-radius: 5px 0 0 5px;
  border: none;
  padding: 10px;
  font-size: 16px;
  width: 70%;
  height: 42px;
}

#submit {
  border-radius: 0 5px 5px 0;
  padding: 10px;
  font-size: 16px;
  width: 5em;
  cursor: pointer;
}

Ambos elementos obtienen los mismos estilos para padding y font-size.

El elemento #search obtiene un 70% de width, así como un height de 42px y ningún border.

El elemento #submit obtiene un width de 5em junto con un estilo cursor: pointer que hace que la flecha se convierta en una mano que señala mientras se desplaza sobre este elemento.

Guardemos styles.css y volvamos a renderizar la página:

A continuación, vamos a aplicar CSS Flexbox para centrar fácilmente el contenido de nuestra aplicación meteorológica.

Selecciona los elementos #weather-wrapper y #weather-search y agrega lo siguiente:

#weather-wrapper {
  display: flex;
}

#weather-search {
  display: flex;
}

Esto convierte los elementos #weather-wrapper y #weather-search en contenedores flexibles. Por lo tanto, sus elementos secundarios se convierten en elementos flexibles que podemos alinear con ciertas propiedades.

Actualiza estos elementos con lo siguiente:

#weather-wrapper {
  display: flex;
  flex-direction: column;
  place-items: center;
  justify-content: space-between;
}

#weather-search {
  display: flex;
  width: 50%;
}

Por defecto, los elementos flex se muestran en una fila. Pero cambiamos esto para el elemento #weather-wrapper estableciendo flex-direction en column.

Para centrar los elementos, utilizamos la propiedad place-items. También hicimos que el elemento #weather-search tuviera la mitad del ancho (50%) de su elemento padre #weather-wrapper.
